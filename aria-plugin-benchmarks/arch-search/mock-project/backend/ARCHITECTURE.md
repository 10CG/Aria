# Backend 架构总览

> **文档层级**: L0 架构文档
> **模块**: Backend 服务端
> **最后更新**: 2026-03-13

## 架构概览

后端采用微服务架构，支持 AI 功能的核心服务集合。

### 核心模块

#### 1. AI 系统
负责智能功能和记忆系统管理。
- **AI 记忆系统**: 用户历史上下文管理
- **自然语言处理**: 对话理解与生成
- **智能推荐**: 个性化内容推荐

```bash
# AI 系统文档路径
└── backend/docs/architecture/ai-memory.md
```

#### 2. API 网关
统一入口，处理路由和认证。
- **RESTful API**: 标准化接口设计
- **GraphQL**: 复杂查询支持

#### 3. 数据层
数据持久化和检索系统。
- **主数据库**: PostgreSQL
- **缓存层**: Redis Cluster
- **向量数据库**: ChromaDB

## 技术栈

### 后端框架
- **Python**: FastAPI (主服务)
- **Dart**: 服务端 Dart (高性能计算)
- **Go**: 微服务 (独立部署)

### 数据存储
- **关系型**: PostgreSQL
- **文档型**: MongoDB
- **向量**: ChromaDB / Pinecone
- **缓存**: Redis

### AI/ML
- **大模型**: OpenAI GPT, Claude
- **向量嵌入**: Sentence-BERT
- **本地推理**: Ollama

## 关键设计原则

### 1. 模块化设计
每个服务独立部署，通过 API Gateway 通信。

### 2. 数据一致性
- 最终一致性模型
- 事件驱动同步

### 3. 可观测性
- 分布式追踪
- 实时监控
- 日志聚合
