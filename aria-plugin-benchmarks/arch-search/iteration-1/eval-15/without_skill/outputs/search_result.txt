### 主要发现

#### 1. 同步架构概述
- **离线优先**: 所有操作优先本地存储
- **自动同步**: 网络恢复后自动同步
- **冲突解决**: 基于时间戳的自动合并

#### 2. 同步流程
```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  Local   │────▶│  Sync    │────▶│  Remote  │
│  Store   │     │  Engine  │     │   API    │
└──────────┘     └──────────┘     └──────────┘
     ▲                │                 │
     └────────────────┴─────────────────┘
                   Sync Queue
```

#### 3. 冲突解决策略
- **Last-Write-Wins (LWW)**: 基于服务器时间戳决定
- **三向合并 (3-Way Merge)**:
  1. 获取 base (共同祖先)
  2. 计算 local diff 和 remote diff
  3. 合并 diffs，自动解决非冲突部分
  4. 冲突部分标记为需要人工处理

#### 4. 同步状态枚举
```dart
enum SyncStatus {
  synced,      // 已同步
  pending,     // 待同步
  conflict,    // 存在冲突
  error,       // 同步失败
}
```
