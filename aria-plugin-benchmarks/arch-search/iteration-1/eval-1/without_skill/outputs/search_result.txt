## 关键发现

### 认证架构概览

认证功能采用 JWT Token 认证机制：

1. **认证服务** (`mobile/src/services/auth/auth_service.dart`)
   - 用户登录 (login)
   - 用户登出 (logout)
   - Token 刷新 (refreshToken)

2. **Token 管理**
   - AccessToken: 存储在内存中
   - RefreshToken: 使用 flutter_secure_storage 安全存储

3. **安全措施**
   - TLS 1.3 强制加密
   - SSL Pinning 证书锁定
   - ProGuard/R8 代码混淆
   - Root/越狱设备检测

### 网络层集成

认证通过拦截器机制集成到网络层：

1. **认证拦截器** (`AuthInterceptor`)
   - 自动添加 Authorization Bearer Token
   - 在请求前检查并添加 Token

2. **错误处理策略**
   - 401 错误: Token 刷新后重试
   - 其他网络错误: 自动重试机制

### 数据存储

后端数据库设计包含：

1. **用户表 (users)**
   - id (UUID)
   - email (唯一)
   - password_hash (加密存储)
   - 其他用户信息

2. **任务表 (tasks)**
   - user_id 外键关联用户
   - 任务基本信息

### API 端点

从 OpenAPI 契约中发现认证相关端点：

- `POST /auth/login` - 用户登录
- `POST /auth/register` - 用户注册

## 与技能搜索的对比优势

**传统方法的优势**:
1. **全面覆盖**: 自动搜索所有相关文件，不会遗漏
2. **灵活调整**: 可以随时修改搜索关键词和策略
3. **深度控制**: 可以精确控制搜索范围和上下文

**传统方法的劣势**:
1. **需要更多指导**: 需要明确知道要搜索哪些关键词
2. **耗时更长**: 需要多次迭代和调整
3. **需要专业知识**: 需要理解项目结构和术语

**观察**: 基线测试展示了传统搜索方法的系统性，但依赖明确的搜索词。而 arch-search 技能可能通过理解语义更快速定位核心信息。
