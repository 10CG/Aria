## 发现的认证相关文档

### 1. 安全架构文档 (Primary)
**路径**: `mobile/docs/architecture/security.md`

**认证实现**:
- JWT Token 认证
  - AuthService 类提供 login/logout/refreshToken 功能
  - TokenPair 数据结构包含 accessToken 和 refreshToken
- Token 存储策略
  - AccessToken: 内存中，自动刷新
  - RefreshToken: 使用 flutter_secure_storage 安全存储
- 安全措施
  - TLS 1.3 强制加密
  - SSL Pinning 防止 MITM 攻击
  - 代码混淆（ProGuard/R8）
  - Root 检测

**相关代码文件**:
- `mobile/src/services/auth/auth_service.dart`
- `mobile/src/services/auth/token_manager.dart`
- `mobile/src/middleware/auth_middleware.dart`

### 2. API 契约文档
**路径**: `shared/contracts/openapi/api.yaml`

**认证接口**:
- POST /auth/login - 用户登录
  - 使用 LoginRequest schema
  - 包含 email 和 password 字段
- POST /auth/register - 用户注册

**结构定义**:
```yaml
LoginRequest:
  type: object

    email: string
    password: string
```

### 3. 网络架构文档
**路径**: `mobile/docs/architecture/network.md`

**认证拦截器**:
- AuthInterceptor 类
- 自动添加 Authorization header: `Bearer {token}`
- 处理 401 错误时刷新 Token

**代码示例**:
```dart
class AuthInterceptor extends Interceptor {
  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) {
    final token = getAccessToken();
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    handler.next(options);
  }
}
```

## Token 使用估计

**搜索过程 token 消耗**:
- 文件列表扫描: ~500 tokens
- Grep 搜索: ~200 tokens
- 读取 9 个文件: ~3,000 tokens
- 生成报告: ~300 tokens
**总计**: ~4,000 tokens

**对比 arch-search skill**:
- 技能版本使用结构化搜索，token 消耗约 1,000-1,500 tokens
- 传统方法消耗约 3 倍的 tokens
- 技能版本提供更精确的代码位置和依赖关系
