### 为什么这样分组？

```
Commit 1 (backend)    ← 独立可测试、可单独回滚
    ↓
Commit 2 (frontend)   ← 依赖 Commit 1 的 API 契约
    ↓
Commit 3 (cleanup)    ← 与功能无关，单独隔离
```

**核心原则：**
- 每个 commit 对应一个**独立关注点**（backend / frontend UI / cleanup）
- 后端先提交 → 前端提交时可引用已稳定的 API 行为
- 清理提交永远不与功能提交混合，方便 `git bisect` 和 code review
