### Issues Found

**1. §3.3 — Missing API endpoints** (omissions, not errors)

These routes exist in `routes.js` but are not documented:
- `POST /api/backups/incremental`
- `GET /api/backup/status`
- `POST /api/backup/force-full`
- `POST /api/backup/force-incremental`
- `POST /api/todos/:todoId/tags/:tagId` — add tag to todo
- `DELETE /api/todos/:todoId/tags/:tagId` — remove tag from todo

**2. §5.2 — todos table missing columns**

`todos` schema omits two columns that exist in the DB:
- `partition_date TEXT`
- `updated_at TIMESTAMP DEFAULT NULL`

**3. §5.1 — ER diagram is misleading on subtasks**

The diagram shows `todos 1──N subtasks (自引用 parentId)`. In reality there are **two** separate mechanisms:
- Self-referential `parentId` column on `todos` itself
- A separate `subtasks` table (`parentId`, `childId`) as a junction table

**4. §3.4 — Undocumented key files**

Several significant backend files not mentioned:
- `backend/src/aiService.js` — AI service layer
- `backend/src/auth/profileController.js` / `profileRoutes.js`
- `backend/src/database/` sub-modules: `backupManager.js`, `cacheManager.js`, `statsManager.js`, `exportManager.js`, `databaseMonitor.js`, `transactionManager.js`, etc.
