# Brainstorm - Requirements Clarification

> **Mode**: Requirements Clarification
> **Input**: "优化系统性能" (vague requirement)
> **Goal**: Extract specific, measurable requirements

---

## 需求模糊性分析

「优化系统性能」是一个非常模糊（ambiguous）的需求，无法直接转化为可执行的任务。在开始之前，我们需要通过澄清问题将其具体化。

**不明确的维度**:
- 哪些方面的性能？（响应时间、吞吐量、内存、CPU？）
- 优化哪些具体功能或接口？
- 当前基线是多少？
- 目标值是多少？
- 是否有 deadline？

---

## Round 1: 性能指标澄清

**核心问题**:

您说的「性能优化」具体指哪个**性能指标**最重要？

```
选项参考（可多选）:
A) 响应时间 (API response time) - 例如 P99 < 200ms
B) 吞吐量 (throughput) - 例如 > 1000 RPS
C) 内存占用 - 例如 < 512MB per instance
D) 数据库查询时间 - 例如 < 50ms per query
E) 页面加载时间 - 例如 FCP < 1.5s
```

---

## 澄清问题清单（将逐一确认）

待您回答 Round 1 后，我们将依次确认以下内容：

1. **Round 1** (当前): 具体性能指标是什么？
2. **Round 2**: 当前基线数据是多少？（现在的性能是多差？）
3. **Round 3**: 目标值是多少？（优化到什么程度算完成？）
4. **Round 4**: 优化范围是哪些模块/API？
5. **Round 5**: 是否有可测量的验收标准？

---

## 已识别的可测量指标示例

一旦澄清完成，需求应该形如:

> "将 `/api/search` 接口的 P99 响应时间从当前 **850ms** 优化到 **200ms** 以内，在高峰期 500 RPS 下稳定运行。"

这是**可测量**的需求，有明确的基线（850ms）、目标（200ms）和场景（500 RPS 高峰期）。
