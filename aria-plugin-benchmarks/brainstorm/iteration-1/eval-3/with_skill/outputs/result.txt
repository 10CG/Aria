## 各选项深度分析

### PostgreSQL
- **优势**: ACID 事务保证，丰富的查询能力，成熟的生态，JSON 支持
- **劣势**: 水平扩展较复杂，大规模分片需额外工具
- **适用场景**: 需要强一致性的业务数据（订单、用户、财务）

### MongoDB
- **优势**: 文档模型灵活，水平扩展原生支持，适合非结构化数据
- **劣势**: 事务支持有限（4.0 后改善），最终一致性风险
- **适用场景**: 内容管理、日志、变化频繁的数据结构

### Redis
- **优势**: 极高读写性能（10w+ QPS），内置过期、pub/sub、缓存
- **劣势**: 数据持久化有限，不适合主数据库，内存成本高
- **适用场景**: 缓存层、会话存储、实时计数、消息队列

---

## 结论与推荐

**推荐方案**: PostgreSQL 作为主数据库 + Redis 作为缓存层

**理由**:
1. 业务数据需要强一致性 → PostgreSQL 最优（得分 5/5）
2. 高并发读请求用 Redis 缓存分担 → 弥补 PostgreSQL 性能短板
3. MongoDB 在此场景无明显优势，引入会增加技术栈复杂度

> 如果数据结构极不稳定或日志类场景，可考虑 MongoDB 替代 PostgreSQL。
