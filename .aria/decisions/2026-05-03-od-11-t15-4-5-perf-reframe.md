但 T15.4/T15.5 验收 D 原文 (proposal §七 + tasks.md T15.5):

**问题**: M1 demo_002_p50 是 M1 baseline 完整 cycle (issue → Layer 1 dispatch → Layer 2 容器路径 → image build → runner exec → PR 生成 → merge)。M2 Layer 1 cycle 终结于 `S4_LAUNCH` HTTP 500 (因 `aria-layer2-runner` parameterized job 不存在 — AD-M2-9 §风险 #1 显式预期)，**不含 Layer 2 任何实际工作**。Wall-clock 时间根本无法比较：

| 阶段 | M1 baseline | M2 实测 (T15.3) |
|---|---|---|
| Layer 1 状态机 | N/A (M1 是手动 dispatch) | ~3s (S0→S4_LAUNCH HTTP) |
| Nomad dispatch HTTP | ~1s | ~3s (含 alloc resolve poll) |
| Layer 2 image build | ~30s | **N/A (jab 不存在)** |
| Layer 2 runner exec | ~120s | **N/A** |
| PR + merge | ~20s | **N/A** |
| **Total** | **~170s** | **~6s** (S_FAIL early) |

M2 6s vs M1 170s × 1.5 = 255s — 表面上 M2 远超过 M1 但实际 M2 没干 Layer 2 的工作, 数字毫无意义。

## 选项

### Option A: Reframe 验收 D 为 "M2 pipeline overhead baseline"

T15.4 收集 metric: 单 issue `S0_IDLE → S4_LAUNCH HTTP dispatch wall` p50 (~3-6s)
- 不与 M1 比，作为 M2 *pipeline overhead* 新基线
- M3 Layer 2 wired 后再补 cycle p50 vs M1

**优点**: 验证 Layer 1 状态机本身 perf 可接受 (cron tick 成本 + LLM 调用 + DB 写入)
**缺点**: 验收 D 文字大改 (proposal + tasks.md), 需要 spec patch

### Option B: Defer T15.4/T15.5 to M3 (carryover)

不在 M2 收集 perf metrics, 直接标 T15.4/T15.5 deferred to US-023 (M3) Spec
- M3 Layer 2 wired 后, 加 perf cell 跑完整 cycle
- M2 验收只剩 验收 A (≥10 dispatched, 已 PASS) + 验收 C (cron 自动化) + 其他

**优点**: M2 sign-off 不阻塞, scope clean
**缺点**: 验收 D 字面 unmet → owner 须显式 waive

### Option C: Waive 验收 D for M2, 标 "M2 partial — Layer 2 perf carryover"

接受 M2 partial sign-off:
- 验收 A ✓
- 验收 D ✗ (carryover to M3, owner explicit waiver)
- M2 Report (T16.4) 显式记录此 carryover

**优点**: 最小 scope 改动, 不需 reframe spec, 不需 schema-level perf 测量
**缺点**: PRD M2 baseline 非完整闭合, owner 需重申 OD-8 156h 容差

### Option D: Deploy stub `aria-layer2-runner` parameterized job → 跑完整 cycle

工时 +6-12h (per `.aria/decisions/2026-05-03-t1-7-t15-2-deploy-debugging.md` Path B caveat "S5/S6/S7/S8 likely have own latent gaps")
- 实质上是把 M3 部分前置到 M2

**优点**: 真正与 M1 比
**缺点**: scope creep (相当于把 M3 alloc_provider + Layer 2 job + S5-S8 wiring 全做), OD-8 156h 容差被严重消耗

## 推荐

**Option C (Waive + carryover)** —

- 最小 scope, 不破坏 spec
- M2 demo 已实证 Layer 1 自动化 (验收 A)
- T15.4/T15.5 → M3 自然落地 (Layer 2 wired 后)
