## 审计结论 (Round 1-3 合并)

### Decisions (6 项, 全票共识)

1. D1: 双模式 (convergence/challenge) 分离
2. D2: 默认关闭 + opt-in
3. D3: 配置化分组, 非运行时判断
4. D4: max_rounds=5 硬上限保护
5. D5: 审计报告持久化到 .aria/audit-reports/
6. D6: adaptive 按 OpenSpec Level 映射审计模式

### Issues (严重 — 4 项, 阻塞实现)

1. **I1a: 收敛判定比较算法缺失** — 需定义结构化比较键 {severity, category, scope, summary}，基于字段集合相等判定，不依赖自由文本
2. **I13: 汇总引擎未定义** — 多 Agent 并发输出的合并策略（投票/优先级/冲突解决）和去重算法需明确
3. **I15: Severity/Verdict 体系缺失** — 新系统输出 decisions/issues/risks 无 severity 字段，旧消费方依赖 audit_verdict: PASS/FAIL，接口断裂
4. **I3+: challenge 模式数据传递 schema 缺失** — 讨论组提案/反对意见列表/全员合并输入的结构化格式，以及中间产物到最终输出的映射

### Issues (重要 — 14 项)

5. I1b: challenge 模式中"全票 PASS"定义不清 — 挑战组沉默是否等价 PASS
6. I2: challenge 收敛条件改为"所有反对意见 resolved"而非"为空"
7. I4: mode=adaptive 与 checkpoints 手动配置优先级规则未定义
8. I5: mid_implementation 触发条件/阈值/计量单位/触发主体全部缺失
9. I6: experiments.agent_team_audit → audit.* 迁移路径 + config-loader 字段验证规则更新
10. I7: T7 需覆盖 6+ 被修改 Phase Skills 的回归 AB 测试
11. I8: T1 估时 8-10h→14-18h, T2 估时 4-6h→8-12h
12. I9: Success Criteria 需定义测试集/样本量/评判方法
13. I10: Target Version v1.2.0 → v1.9.0 (当前 plugin v1.8.0)
14. I11: 依赖图多处与文字描述不一致, T4/T5 可并行解耦
15. I16: 人工介入具体协议未定义 (暂停/摘要/强制收敛/resume)

### Issues (中等/一般 — 7 项)

### Risks (高 — 3 项)

1. R1: 收敛振荡 — 需区分噪声振荡(delta<threshold→自动取最后轮)和真实分歧(→人工介入)
2. R2: 未收敛降级策略 — 三路径: 用户 override 继续 / 调整 max_rounds 重试 / 降级到单轮
3. R3: agent-team-audit → audit-engine 迁移 — 推荐组合关系(audit-engine 编排, agent-team-audit 退化为单轮执行引擎)

### Risks (中/低 — 7 项)

4. R4: Token 消耗 — 最坏 7x5x4=140 次调用, 需预算估算和 token_budget 配置
5. R5: 同一模型对抗性有限
6. R6: 多轮上下文溢出 — 需滚动压缩策略
7. R7: 修改面广 + 回归测试负担
8. R8: brainstorm 输出格式与审计输入不兼容
9. R9: 审计报告长期积累维护负担
