## 6. 评分因子详解

```
┌─────────────────────────────────────────────────────────────────┐
│                      5因子评分系统                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  因子 1: file_count (变更文件数)                                 │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  1-3 文件   → 0 分  │  4-10 文件  → +1 分  │  10+ 文件 → +3 分│  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  因子 2: cross_directory (跨目录变更)                            │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  否 → 0 分                    │  是 → +2 分                 │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  因子 3: task_count (任务数量)                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  1-3 任务   → 0 分  │  4-8 任务   → +1 分  │  8+ 任务 → +3 分│  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  因子 4: risk_level (风险等级)                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  低 → 0 分  │  中 → +1 分  │  高 → +3 分                    │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  因子 5: parallel_needed (并行需求)                              │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  否 → 0 分                    │  是 → +5 分 (强制 L3)       │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │
│  决策阈值:                                                       │
│  ├─ 评分 < 3  → L1 (对话隔离)                                   │
│  ├─ 3 <= 评分 < 6 → L2 (文件系统隔离)                           │
│  └─ 评分 >= 6 → L3 (完全隔离)                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 7. 实际场景示例

### 场景 1: 单文件 Bug 修复

```
任务: 修复 auth.js 中的登录 Bug

评分:
├─ file_count: 1 → 0 分
├─ cross_directory: 否 → 0 分
├─ task_count: 1 → 0 分
├─ risk_level: 低 → 0 分
└─ parallel_needed: 否 → 0 分

总分: 0 → L1 对话隔离

执行:
└─ Fresh Subagent 在主分支直接修改
```

### 场景 2: 新功能开发

```
任务: 添加用户认证功能

评分:
├─ file_count: 6 (auth.js, login.js, test, etc.) → +1 分
├─ cross_directory: 是 (src/ + tests/) → +2 分
├─ task_count: 4 → +1 分
├─ risk_level: 中 → +1 分
└─ parallel_needed: 否 → 0 分

总分: 5 → L2 文件系统隔离

执行:
├─ 创建 Worktree: .git/worktrees/TASK-001-auth
└─ Fresh Subagent 在 Worktree 中开发
```

### 场景 3: 大规模重构

```
任务: 重构整个 API 层

评分:
├─ file_count: 25 → +3 分
├─ cross_directory: 是 → +2 分
├─ task_count: 12 → +3 分
├─ risk_level: 高 → +3 分
└─ parallel_needed: 否 → 0 分

总分: 11 → L3 完全隔离

执行:
├─ 创建 Worktree: ../aria-refactor-api
├─ 独立安装依赖: npm install
└─ Fresh Subagent 在完全隔离环境中开发
```

### 场景 4: 并行多任务

```
任务: 同时开发 3 个独立功能

评分:
├─ file_count: 任意
├─ cross_directory: 任意
├─ task_count: 任意
├─ risk_level: 任意
└─ parallel_needed: 是 → +5 分 (强制 L3)

总分: >= 5 → L3 完全隔离

执行:
├─ Worktree 1: feature/task-001
├─ Worktree 2: feature/task-002
└─ Worktree 3: feature/task-003
   (每个 Worktree 独立依赖和进程)
```

## 8. 隔离级别对比表

| 维度 | L1 对话隔离 | L2 文件系统隔离 | L3 完全隔离 |
|------|------------|----------------|------------|
| **上下文隔离** | ✅ Fresh Subagent | ✅ Fresh Subagent | ✅ Fresh Subagent |
| **文件系统隔离** | ❌ 共享主分支 | ✅ 独立 Worktree | ✅ 独立 Worktree |
| **依赖隔离** | ❌ 共享 | ⚠️ 部分共享 | ✅ 完全独立 |
| **启动速度** | ⚡ 极快 (<1s) | 🚀 快 (2-5s) | 🐢 慢 (30-60s) |
| **磁盘占用** | 💾 0 MB | 💾 ~50 MB | 💾 ~500 MB |
| **回滚能力** | ❌ 无 | ✅ 快速 | ✅ 快速 |
| **并行支持** | ❌ 不支持 | ⚠️ 有限 | ✅ 完全支持 |
| **适用场景** | 简单修复 | 中等功能 | 大规模重构 |
