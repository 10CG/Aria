# US-011: 项目适配能力 — Agent 覆盖度分析 + 配置生成

> **Priority**: MEDIUM
> **Status**: ready
> **Parent PRD**: prd-aria-v1.md
> **Target Version**: v1.13.0
> **Source**: Forgejo aria-plugin#3 (Layer 2 拆分)
> **Depends On**: [US-010](./US-010.md) (STCO description + handoff contract)

## User Story

**作为** 将 Aria 接入新项目的开发者,
**我希望** Aria 能自动分析目标项目的技术栈和工作模式,评估现有 11 个 Agent 的覆盖度,识别能力缺口,并在人工确认后生成项目专属 Agent 配置,
**以便** 新项目接入时 Agent Team 能立即适配项目特性,而非依赖通用角色定义。

## 背景

### 问题来源

1. **Kairos 跨项目 Pilot** (aria-plugin#2, closed): TypeScript/Node.js 项目接入时,backend-architect 不了解项目技术栈约束,建议偏 Python/Go 范式
2. **aria-plugin#3** 原始需求: 项目分析 + Agent 覆盖度评估 + Agent 配置生成
3. **US-010 Agent Team 讨论** (2026-04-11): 4/4 共识拆分为 Layer 1 (质量强化, done) + Layer 2 (项目适配, 本 US)

### PromptX 研究启发 (2026-04-11)

| PromptX 概念 | Layer 2 借鉴 |
|-------------|-------------|
| Nuwa 角色创造者 | 项目分析后 AI 生成 Agent 配置,需人工确认 |
| 技术栈模板库 | 按技术栈 (Node.js/Python/Go/Flutter) 提供最小 Agent 模板集 |
| RoleX 生命周期 | **不采纳** — Agent 保持无状态 (D3, US-010) |

### 设计决策 (Agent Team 讨论已收敛)

| ID | 决策 | 理由 | 来源 |
|----|------|------|------|
| D1 | 生成物放项目级 `.aria/agents/`, 非 aria-plugin 全局 | 项目级 Agent 生命周期跟随项目, 不污染全局 | Tech Lead |
| D2 | 缺口报告 → 人工确认 → 生成 (禁止跳过确认) | 规范先行原则: 无 OpenSpec 背书的 Agent 不应自动生效 | Backend Architect |
| D3 | 按技术栈分组的最小模板集 (非预设角色库) | 模板是起点不是终点, 避免"模板越多越好"陷阱 | Backend Architect |
| D4 | agent-router 需动态感知项目级 Agent (routing-hints) | 生成的 Agent 需被路由发现, 否则等于没用 | Backend Architect |
| D5 | 生成后需人工审阅 + 质量验证 | 劣质 description 会降低整体路由准确率 | QA |

## Acceptance Criteria

1. 新 Skill `project-analyzer`: 扫描目标项目技术栈/目录/工作流 → 输出结构化项目画像
2. 新 Skill `agent-gap-analyzer`: 对比项目画像 vs 现有 11 Agent 能力矩阵 → 输出覆盖度报告 + 缺口清单
3. 新 Skill `agent-creator`: 基于缺口清单 + 技术栈模板 → 人工确认后生成 `.aria/agents/<name>.md`
4. 生成的 Agent 遵循 STCO 模式 (Scope-Trigger-Contract-Output, US-010 定义)
5. agent-router 能自动发现并路由到 `.aria/agents/` 下的项目级 Agent
6. 生成流程包含人工确认步骤 (缺口报告 → 确认 → 模板选择 → 生成)

## Scope

- 新增: 2-3 个 Skill (project-analyzer, agent-gap-analyzer, agent-creator 或合并)
- 修改: agent-router (动态扫描 `.aria/agents/`)
- 新增: 技术栈模板集 (Node.js/Python/Go/Flutter 各一最小模板)
- 不修改: 现有 11 个 Agent
- 不新增: 全局 Agent (仅项目级)
