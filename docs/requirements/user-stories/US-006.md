# US-006: state-scanner 项目级自定义健康检查

> **Priority**: MEDIUM
> **Status**: done
> **Parent PRD**: prd-aria-v1.md
> **Target Version**: v1.4.0
> **Source**: Forgejo Issue #4

---

## User Story

**作为** 使用 Aria 方法论的项目维护者，
**我希望** 在 state-scanner 中定义项目特有的健康检查项（如 benchmark 时效性、文档同步状态、依赖审计），
**以便** AI 在扫描项目状态时能感知项目级的健康指标，给出更精准的工作流推荐。

## 验收标准

- [ ] state-scanner 新增阶段 1.11，读取 `.aria/state-checks.yaml` 配置
- [ ] 支持执行用户定义的 shell 命令，解析 exit code 判定 pass/fail
- [ ] 支持 severity 三级分类 (info/warning/error)，联动阶段 2 推荐引擎
- [ ] 单检查超时 15s，总超时 60s，超时不阻塞主流程
- [ ] `fix` 命令仅作为建议展示，不自动执行
- [ ] 配置文件不存在时静默跳过，格式错误时输出警告并跳过
- [ ] state-scanner 输出中新增自定义检查区块
- [ ] AB benchmark delta 为正值 (Rule #6)

## 技术备注

- 配置独立存放于 `.aria/state-checks.yaml`，不纳入 config-loader
- 串行执行检查命令，与 hooks.json 信任模型一致，不做沙箱
- 检查结果通过 `context.custom_checks` 传递给 workflow-runner
- Aether benchmark freshness 作为验收用例
