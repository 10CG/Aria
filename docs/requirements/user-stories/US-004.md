# US-004: 十步循环自动审计系统

> **Priority**: MEDIUM
> **Status**: pending
> **Parent PRD**: prd-aria-v1.md
> **Target Version**: v1.2.0

---

## User Story

**作为** Aria 方法论的使用者，
**我希望** 十步循环在关键环节自动触发多 Agent 审计（循环审计或挑战审计），
**以便** 在 AI 协作流程中获得质量保障、决策验证和多视角共识，减少遗漏和偏差。

## 验收标准

- [ ] 支持 7 个检查点：post_brainstorm, post_spec, post_planning, post_implementation, mid_implementation, pre_merge, post_closure
- [ ] 支持两种审计模式：convergence（循环审计）和 challenge（挑战审计）
- [ ] 支持 adaptive 模式：Level 1 跳过, Level 2 convergence, Level 3 challenge
- [ ] 收敛判定：结论集合无变化 + 全票 PASS 双重验证
- [ ] max_rounds 保护（默认 5），超限输出未收敛警告
- [ ] 挑战审计按检查点动态分组讨论组/挑战组
- [ ] config-loader 扩展：无配置时自动分析项目特征生成审计配置
- [ ] 审计报告输出到 .aria/audit-reports/

## 技术备注

- 扩展现有 agent-team-audit 实验功能，从单轮升级为多轮收敛
- config-loader 需新增项目特征分析 + 配置自动生成能力
- 需要新增 audit-engine Skill（审计引擎核心）
- 7 个检查点需集成到对应的 Phase Skills 中
