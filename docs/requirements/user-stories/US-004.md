## User Story

**作为** Aria 方法论的使用者，
**我希望** 十步循环在关键环节自动触发多 Agent 审计（循环审计或挑战审计），
**以便** 在 AI 协作流程中获得质量保障、决策验证和多视角共识，减少遗漏和偏差。

## 验收标准

- [x] 支持 7 个检查点 — T4 升级 3 个已有 + T5 新增 4 个
- [x] 支持两种审计模式 — audit-engine SKILL.md convergence + challenge 双模式
- [x] 支持 adaptive 模式 — state-scanner v2.7.0 复杂度传递 + config adaptive_rules
- [x] 收敛判定 — 四元组比较 + 全票 PASS + 振荡检测
- [x] max_rounds 保护 — 默认 5, 未收敛三路径降级策略
- [x] 挑战审计动态分组 — 配置化 teams 分组, 11 Agents x 7 检查点
- [x] config-loader 扩展 — 旧配置兼容映射 + 审计默认值 + 配置示例
- [x] 审计报告输出 — .aria/audit-reports/{checkpoint}-{timestamp}.md, 含 verdict

## 技术备注

- 扩展现有 agent-team-audit 实验功能，从单轮升级为多轮收敛
- config-loader 需新增项目特征分析 + 配置自动生成能力
- 需要新增 audit-engine Skill（审计引擎核心）
- 7 个检查点需集成到对应的 Phase Skills 中
