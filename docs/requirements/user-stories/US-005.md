# US-005: 项目进度看板与自动开发闭环

> **Priority**: HIGH
> **Status**: pending
> **Parent PRD**: prd-aria-v1.md
> **Target Version**: v1.3.0

---

## User Story

**作为** 使用 Aria 方法论的团队成员（包括非 CLI 用户），
**我希望** 通过一个可部署的 Web 看板查看项目进度、提交 issue、并由 AI 自动分析和实施 issue 对应的开发，
**以便** 实现从需求输入到代码交付的全自动闭环，降低人工干预。

## 验收标准

### Phase 1: 完整看板
- [ ] `/aria:dashboard` 生成单文件自包含 HTML 看板
- [ ] 展示 UPM 进度（Phase/KPI/Cycle）
- [ ] 展示 User Story 三列看板（TODO/进行中/已完成）
- [ ] 展示 OpenSpec 列表（活跃 + 归档 + 创建→归档耗时）
- [ ] 展示审计报告历史（轮次/收敛/verdict）
- [ ] 展示 AB benchmark 摘要（delta 趋势）
- [ ] 数据源全部来自已有 Markdown/JSON 文件，零外部依赖

### Phase 2: Issue 提交
- [ ] 看板页面包含 issue 提交表单（标题/描述/优先级）
- [ ] Git 原生模式：issue 存为 `.aria/issues/ISSUE-{N}.md`
- [ ] 可选 API 模式：通过 GitHub/Forgejo API 创建 issue
- [ ] 看板可部署为长期运行的 Web 服务

### Phase 3: 心跳 Agent
- [ ] AI 定时扫描新 issue（心跳机制）
- [ ] 自动分析 issue 复杂度（Level 1/2/3）
- [ ] 全自动执行十步循环（spec→plan→develop→test→commit→PR）
- [ ] 完成后更新 issue 状态 + 看板数据自动刷新

## 技术备注

- Phase 1 基于 SilkNode progress-dashboard 提炼，解析逻辑已验证
- Phase 2 需要将静态 HTML 升级为可部署 Web 应用
- Phase 3 依赖 Claude Code `/schedule` (remote agent) 能力
- Issue 存储适配器设计支持 Git 原生 + API 双模式
